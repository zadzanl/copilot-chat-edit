# Copilot Chat Porter v0.1.0 ‚Äî Product Requirements Document

## Overview

Three new capabilities for the Copilot Chat Porter extension:

1. **External script access**: Beyond the existing LM Tool channel, expose a URI Handler so plain bash scripts in VS Code Hooks can trigger extension functionality without going through an LLM
2. **Attachment saving**: Optionally copy externally-referenced files alongside the exported JSON, making the export self-contained and portable
3. **Command Palette entry**: Trigger export/import directly via `Cmd+Shift+P`

---

## Path Rules

This section defines the path resolution rules for all parameters. These apply to both the LM Tool and URI Handler channels.

### `outputPath` (export file path)

- Accepts absolute and relative paths
- Relative paths are resolved from the workspace root
- Optional. When omitted, auto-generates a timestamped filename in the workspace root

### `inputPath` (import file path)

- Accepts absolute and relative paths
- Relative paths are resolved from the workspace root
- Required

### `attachmentsDir` (attachments directory)

- Accepts relative paths only, resolved from the exported JSON file's parent directory
- Passing an absolute path should produce an error
- Optional. When omitted, no attachments are copied

Example: exporting to `exports/chat.json` with `attachmentsDir=attachments` saves attachments to `exports/attachments/`.

### `callback` (synchronous callback path, export via URI Handler only)

- Accepts absolute paths only
- Optional. When omitted, no result feedback is provided
- Must be URL-encoded in URI Handler calls (e.g., spaces encoded as `%20`)

## Feature 1: External Script Access

### Motivation

The extension can currently only be invoked via Language Model Tools, which requires going through an LLM. Copilot Hooks (UserPromptSubmit, etc.) run OS-level bash scripts that cannot directly call VS Code extension commands. A trigger channel outside the LLM is needed.

### Communication Mechanism

Uses the VS Code URI Handler (`vscode.window.registerUriHandler`).

- The extension registers a URI Handler listening for requests in the format `vscode://Vizards.copilot-chat-porter/<action>?<params>`
- External scripts trigger it via `code --open-url "vscode://Vizards.copilot-chat-porter/..."`
- `activationEvents` must include `"onUri"`

> **Technical research conclusion**: The URI Handler is VS Code's only official external‚Üíextension communication mechanism. Verified that calling `workbench.action.chat.export` via URI Handler during a Stop hook executes successfully ‚Äî no timing issues.

### Supported Actions

- **Export**: `vscode://Vizards.copilot-chat-porter/export?outputPath=<path>&attachmentsDir=<path>&callback=<path>`
- **Import**: `vscode://Vizards.copilot-chat-porter/import?inputPath=<path>`

All query parameters are optional (except `inputPath` for import).

### Optional Synchronous Callback (`callback`)

The URI Handler is inherently fire-and-forget. For synchronous result feedback, pass a `callback` parameter specifying a file path (typically a Named Pipe / FIFO). The extension writes the result JSON to that path once the operation completes.

Result format:

```json
{
  "success": true,
  "outputPath": "exports/chat.json",
  "attachments": [
    { "from": "/Users/xxx/Downloads/report.pdf", "to": "/Users/xxx/project/exports/attachments/report.pdf" }
  ],
  "error": null
}
```

`outputPath` and `attachments` are present on success; `error` is present on failure.

Typical usage (bash):

```bash
PIPE=$(mktemp -u /tmp/porter-XXXXX.pipe)
mkfifo "$PIPE"
code --open-url "vscode://Vizards.copilot-chat-porter/export?outputPath=exports/chat.json&callback=$(python3 -c "import urllib.parse; print(urllib.parse.quote('$PIPE'))")"
RESULT=$(timeout 30 cat "$PIPE")
rm -f "$PIPE"
```

Omitting `callback` means no result feedback ‚Äî suitable for fire-and-forget scenarios like async archiving.

### Constraints

- The external access channel shares the same core logic as the LM Tool; behavior is identical

## Feature 2: Attachment Saving

### Motivation

The exported conversation JSON contains file references (`fsPath`) pointing to local machine paths (e.g., `~/Downloads/report.pdf`). These external files may be deleted or moved at any time, invalidating the references. Copying them alongside the export makes the artifact portable.

### Trigger

Opt-in, controlled by the `attachmentsDir` parameter:

- **`attachmentsDir` omitted**: no attachments are copied
- **`attachmentsDir` provided**: attachments are copied to the specified directory

No separate toggle is needed ‚Äî the presence of `attachmentsDir` itself signals "copy attachments".

- **LM Tool**: `attachmentsDir: string` added to `inputSchema` (optional)
- **URI Handler**: passed as a query parameter `attachmentsDir=<path>`

### Which Files Are Copied

Only files **outside the workspace**. The criterion: a file's absolute path does not start with any open `workspaceFolder` path.

Files inside the workspace are already part of the project and don't need duplicating.

### External File Discovery

Uses structural URI object detection. Scans the exported JSON for URI objects (objects containing both `$mid` and `scheme: "file"` fields), extracting the absolute path from `fsPath`; falls back to `path` if `fsPath` is absent.

> This detection is based on VS Code core's `URI.toJSON()` serialization format (not Copilot Chat‚Äìspecific JSON structure), so it's highly stable.

### File Naming

Preserves the original filename. On name collision, appends a numeric suffix (e.g., `report-1.pdf`).

### Path Rewriting

After copying attachments, performs bulk string replacement on the serialized JSON string:

For each copied file, builds a mapping:
- `oldAbsolutePath` ‚Üí `newAbsolutePath`
- `oldFileUri` (`file:///...`, URL-encoded) ‚Üí `newFileUri`

Then executes `replaceAll` on the JSON string, replacing all matches in one pass.

> **Design decision**: Rather than targeting specific JSON field names (`fsPath`, `external`, `path`, `fullName`, etc.) for point replacements, we match on values and replace globally. This way, even if Copilot Chat changes its JSON structure, adds fields, or renames them, the rewriting still works as long as the path value appears somewhere in the JSON.

### Edge Cases

- Original file no longer exists (user deleted it): skip the file, log a warning, don't interrupt the overall export
- Target location already has a file with identical content (determined by byte comparison): skip the copy
- Very large files: no size cap; the filesystem handles it

## LM Tool Changes

### exportChatHistory

#### inputSchema

New `attachmentsDir` field:

```json
{
  "type": "object",
  "properties": {
    "outputPath": {
      "type": "string",
      "description": "Path to save the exported JSON file. Accepts absolute path or relative path (resolved from workspace root). If omitted, auto-generates a timestamped filename."
    },
    "attachmentsDir": {
      "type": "string",
      "description": "If provided, copies externally-referenced files (outside workspace) into this directory. Relative path only, resolved from the exported JSON file's directory. If omitted, no attachments are copied."
    }
  }
}
```

#### modelDescription

Update `modelDescription` to describe the `attachmentsDir` parameter so the LLM can correctly understand and use the attachment saving feature. Should cover: when to pass `attachmentsDir` (when the user wants to preserve externally-referenced files), and how the relative path is resolved.

#### Return Value

When `attachmentsDir` is provided, the return value includes attachment sync results in addition to the export path: each copied file's original path and its new path.

Example return text:
```
Chat history exported to: exports/chat.json
Attachments synced:
  /Users/xxx/Downloads/report.pdf ‚Üí /Users/xxx/project/exports/attachments/report.pdf
  /Users/xxx/Downloads/image.png ‚Üí /Users/xxx/project/exports/attachments/image.png
```

### importChatHistory

#### inputSchema

Updated `inputPath` description to reflect absolute path support:

```json
{
  "type": "object",
  "required": ["inputPath"],
  "properties": {
    "inputPath": {
      "type": "string",
      "description": "Path to the chat history JSON file to import. Accepts absolute path or relative path (resolved from workspace root)."
    }
  }
}
```

No other behavioral changes.

## Feature 3: Command Palette Entry

### Motivation

Provide a standard VS Code interaction entry point. Users can trigger export/import directly via `Cmd+Shift+P`, without the LLM and without scripts.

### Registered Commands

- `copilot-chat-porter.export` ‚Äî Export the current conversation
- `copilot-chat-porter.import` ‚Äî Import a conversation

### Interaction Flow

**Export**:
1. Show a save file dialog; user picks the export path (default: timestamped filename)
2. Show an input box prompting for `attachmentsDir` (leave empty to skip attachment copying)
3. Call core export logic

**Import**:
1. Show an open file dialog; user selects a JSON file
2. Call core import logic

### Constraints

- Shares the same core logic as the LM Tool and URI Handler
